Task: Transcript Summarization and Translation service
 
Your task is to implement a simple service, where doctors can summarize text and translate it as needed. Doctors can use it to summarize discussion transcripts with patients, or any other textual data. 
Features
The application should meet the following requirements.
Must haves:
● User has the possibility to paste text into text field
● There is a button, pressing it will summarize the text into single paragraph of text
● Additionally, user can choose the language of the summary text, minimal two different languages
● If user refreshes the page, the summary text stays on, until new text is summarized
 
Optional features:
● Upload a file and read the text contents to summarize
● Additional personalization of the summary text (more languages, different styling (bulletpoints vs paragraphs), tonality)
● Ability to regenerate a summary text
● Multiple different type of summaries from one transcript
Things to note
The guidelines are that the task is to take between 4-6 hours, with a maximum cutoff time of eight hours. Please honor this timeline.
Deliverable
A github repository or such, optionally have the app running in some cloud service.
